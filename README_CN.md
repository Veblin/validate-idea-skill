# Validate Idea / 想法拷问（多专家版）

一个 Claude Code 技能：启动 5 个并行的专家子 Agent，在你投入时间之前对创业/产品想法进行全面拷问。

## 工作原理

```
用户描述想法
      │
      ▼
┌─ 主 Agent ─────────────────────────────────────┐
│ 1. 收集信息（5 个澄清问题）                       │
│ 2. 并行启动 5 个专家                             │
│ 3. 交叉验证 → 判定                              │
└─────────────────────────────────────────────────┘
      │         │         │         │         │
      ▼         ▼         ▼         ▼         ▼
  🎯 风险     👂 客户     🔍 竞品     💰 投资人   🧪 价值主张
  猎人       真相调查官   穷举师     质询官      拷问官
```

## 五位专家

| 专家 | 框架 | 关注点 |
|------|------|--------|
| 🎯 风险猎人 | Riskiest Assumption Test (RAT) | 找出前 3 个致命假设 + 设计 ≤2 周/≤$100 的验证实验 |
| 👂 客户真相调查官 | 《The Mom Test》Rob Fitzpatrick | 写出不提及产品、只追问过去行为的客户访谈问题 |
| 🔍 竞品穷举师 | 穷举式多层搜索 | 直接/间接/替代方案/平台风险/AI 替代 — 不存在没有竞品的赛道 |
| 💰 投资人质询官 | 天使投资尽调 | 止痛药还是维生素？时机？护城河动态？单位经济？创始人匹配度？ |
| 🧪 价值主张拷问官 | 《Value Proposition Design》Osterwalder | Jobs-Pains-Gains 拆解 + 10 个通过/不通过的批判性问题 |

## 判定

| 判定 | 含义 |
|:----:|------|
| 🟢 **可推进** | 最高风险假设已识别，有具体的验证实验方案 |
| 🟡 **需收窄** | 砍掉部分功能/用户群/市场，收窄后重新验证 |
| 🔴 **放弃** | 发现致命缺陷 — 这个方向不应该继续 |

## 安装

```bash
mkdir -p ~/.claude/skills/validate-idea
cp SKILL.md ~/.claude/skills/validate-idea/SKILL.md
```

## 使用

```
/validate-idea
```

或者直接描述你的想法 — Claude 会在启动专家前追问澄清问题。

## 语言支持

技能会自动匹配用户的语言。用中文提问，得到中文回复；用英文提问，得到英文回复。

## 灵感来源

- **Riskiest Assumption Test (RAT)** — 找到最先杀死你的假设
- **《The Mom Test》** Rob Fitzpatrick 著 — 如何与客户交谈而不自我欺骗
- **《Value Proposition Design》** Alexander Osterwalder 著 — Jobs-Pains-Gains 框架
- **天使投资尽调框架** — 投资人在开支票前真正会问的问题
